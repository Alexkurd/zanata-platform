Introduction

This is the documentation for the Zanata REST API. It includes detailed information about the resources, content, response codes, etc available to external users.

Authentication

Some of the available resources require authentication and will respond with an UNAUTHORIZED (401) code if not available. To prevent this, you must supply credentials in the form of HTTP headers as indicated in the table below. Additionally, the service might continue to respond with the same code if the authenticated user does not have the right permissions to access the resource.
Header Description
X-Auth-User User name
X-Auth-Token User's API key
